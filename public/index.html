⚡ Gerenciador de Pokémons ⚡

Sistema completo para gerenciar seu time, boxes e itens

Sobre o Projeto

O Gerenciador de Pokémons é um sistema web desenvolvido para treinadores gerenciarem seus Pokémons de forma eficiente e organizada. O sistema permite que cada treinador tenha um time principal com até 6 Pokémons, várias boxes para armazenar Pokémons adicionais, e um inventário completo de itens para uso durante as aventuras.

Funcionalidades Principais

Gerenciamento de Time

  • Manter um time principal com até 6 Pokémons
  • Adicionar e remover Pokémons do time
  • Reorganizar a ordem dos Pokémons no time
  • Visualizar informações detalhadas de cada Pokémon no time

Sistema de Boxes

  • Criar e gerenciar múltiplas boxes de armazenamento
  • Armazenar Pokémons em boxes organizadas
  • Transferir Pokémons entre boxes e o time principal
  • Visualizar o conteúdo de cada box
  • Buscar Pokémons específicos nas boxes

Gerenciamento de Itens

  • Manter um inventário de itens do treinador
  • Adicionar e remover itens do inventário
  • Organizar itens por categoria (Pokébolas, curas, itens especiais, etc.)
  • Controle de quantidade de cada item
  • Usar itens em Pokémons do time

Perguntas do Laboratório

Quais são as principais funcionalidades que o sistema deve oferecer para atender às necessidades dos usuários?

O sistema deve oferecer as seguintes funcionalidades principais para atender às necessidades dos usuários:

  • Gerenciamento de Time: Permitir que o usuário monte e gerencie um time principal com exatamente 6 Pokémons, podendo adicionar, remover e reorganizar os membros do time.
  • Sistema de Boxes: Oferecer múltiplas boxes de armazenamento onde o usuário pode guardar Pokémons além dos 6 do time principal, com capacidade de criar, organizar e gerenciar essas boxes.
  • Transferência de Pokémons: Facilitar a transferência de Pokémons entre o time principal e as boxes, permitindo uma gestão fluida da coleção.
  • Gerenciamento de Itens: Manter um inventário completo de itens do treinador, incluindo Pokébolas, itens de cura, itens especiais, com capacidade de adicionar, remover e usar esses itens.
  • Visualização e Busca: Oferecer interfaces claras para visualizar informações dos Pokémons, buscar Pokémons específicos nas boxes e navegar pela coleção de forma intuitiva.
  • Persistência de Dados: Garantir que todas as informações do usuário (time, boxes e itens) sejam salvas e recuperadas corretamente entre sessões.

Quais características de qualidade (como desempenho, segurança, usabilidade ou escalabilidade) são importantes para o seu sistema desenvolvido?

As seguintes características de qualidade são essenciais para o sistema:

  • Usabilidade: Interface intuitiva e fácil de usar, permitindo que usuários gerenciem seus Pokémons de forma rápida e eficiente, com navegação clara e feedback visual adequado para todas as ações.
  • Desempenho: Sistema responsivo que carrega e processa informações rapidamente, mesmo com grandes coleções de Pokémons. Otimização de consultas e renderização para garantir uma experiência fluida.
  • Segurança: Autenticação e autorização adequadas para garantir que cada treinador acesse apenas seus próprios dados. Validação de entrada para prevenir ataques e proteção contra perda de dados.
  • Escalabilidade: Arquitetura que suporte múltiplos usuários simultâneos e grandes volumes de dados. Capacidade de adicionar novas funcionalidades sem comprometer a performance do sistema.
  • Confiabilidade: Sistema estável que não perde dados e recupera-se adequadamente de erros. Backup regular e mecanismos de recuperação de dados em caso de falhas.
  • Manutenibilidade: Código bem estruturado e documentado que facilita futuras modificações e expansões do sistema.
